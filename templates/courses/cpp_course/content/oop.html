Object-Oriented Programming (OOP) in C++

Introduction: OOP in C++ allows you to model real-world entities using classes and objects, making code reusable, organized, and scalable.

1. Classes & Objects

Class: Blueprint for creating objects (instances).

class Car {
  public:
    string brand;
    int year;
};

int main() {
  Car myCar;
  myCar.brand = "Toyota";
  myCar.year = 2020;
  cout << myCar.brand << " " << myCar.year;
}

2. Access Specifiers

  • public - Accessible from anywhere.
  • private - Accessible only within the class.
  • protected - Accessible in derived classes.
class Student {
  private:
    int age;

  public:
    void setAge(int a) { age = a; }
    int getAge() { return age; }
};

3. Constructors & Destructors

Constructor: Special method called when object is created.

Destructor: Cleans up when object is destroyed.

class Person {
  public:
    Person() { cout << "Constructor called!" << endl; }
    ~Person() { cout << "Destructor called!" << endl; }
};

int main() {
  Person p1;
}

4. Inheritance

Allows a class (child) to inherit properties and methods from another class (parent).

Single Inheritance:

class Animal {
  public:
    void eat() { cout << "Eating..." << endl; }
};

class Dog : public Animal {
  public:
    void bark() { cout << "Barking!" << endl; }
};

int main() {
  Dog d;
  d.eat();
  d.bark();
}

Multilevel Inheritance:

class Animal {
  public:
    void eat() { cout << "Eating..." << endl; }
};

class Dog : public Animal {
  public:
    void bark() { cout << "Bark!" << endl; }
};

class Puppy : public Dog {
  public:
    void weep() { cout << "Weeping..." << endl; }
};

int main() {
  Puppy p;
  p.eat();
  p.bark();
  p.weep();
}

5. Polymorphism

Polymorphism means "many forms" — same function behaves differently based on context.

Compile-Time Polymorphism (Function Overloading):

class Print {
  public:
    void show(int i) { cout << "Integer: " << i << endl; }
    void show(double d) { cout << "Double: " << d << endl; }
};

int main() {
  Print p;
  p.show(5);
  p.show(5.5);
}

Run-Time Polymorphism (Virtual Functions):

class Animal {
  public:
    virtual void sound() { cout << "Some sound" << endl; }
};

class Dog : public Animal {
  public:
    void sound() override { cout << "Bark" << endl; }
};

int main() {
  Animal* a;
  Dog d;
  a = &d;
  a->sound();  // Bark
}

6. Encapsulation

Wrapping data (variables) and code (functions) together while restricting direct access.

class Employee {
  private:
    int salary;

  public:
    void setSalary(int s) { salary = s; }
    int getSalary() { return salary; }
};

int main() {
  Employee e;
  e.setSalary(5000);
  cout << e.getSalary();
}

7. Abstraction

Showing essential features while hiding details. Achieved using abstract classes.

class Shape {
  public:
    virtual void draw() = 0;  // Pure virtual function
};

class Circle : public Shape {
  public:
    void draw() override { cout << "Drawing Circle" << endl; }
};

int main() {
  Circle c;
  c.draw();
}

Real-World Practice Example: Bank System

class BankAccount {
  private:
    string owner;
    double balance;

  public:
    BankAccount(string name, double bal) {
      owner = name;
      balance = bal;
    }

    void deposit(double amount) {
      balance += amount;
    }

    void display() {
      cout << owner << " has $" << balance << endl;
    }
};

int main() {
  BankAccount acc("Alice", 1000);
  acc.deposit(500);
  acc.display();
}
