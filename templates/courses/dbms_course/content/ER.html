Based on the ER diagram, here is the schema design for the database, which defines the structure of the tables and relationships between them.

üìú Table Definitions

1. Student Table

CREATE TABLE Student (
    student_id INT PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100) UNIQUE
);

2. Instructor Table

CREATE TABLE Instructor (
    instructor_id INT PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100) UNIQUE
);

3. Course Table

CREATE TABLE Course (
    course_id INT PRIMARY KEY,
    name VARCHAR(100),
    description TEXT,
    instructor_id INT,
    FOREIGN KEY (instructor_id) REFERENCES Instructor(instructor_id)
);

4. Enrollment Table

CREATE TABLE Enrollment (
    enrollment_id INT PRIMARY KEY,
    student_id INT,
    course_id INT,
    FOREIGN KEY (student_id) REFERENCES Student(student_id),
    FOREIGN KEY (course_id) REFERENCES Course(course_id)
);

5. Schedule Table

CREATE TABLE Schedule (
    schedule_id INT PRIMARY KEY,
    course_id INT,
    day_of_week VARCHAR(10),
    start_time TIME,
    end_time TIME,
    FOREIGN KEY (course_id) REFERENCES Course(course_id)
);

üìã Relationships

  • Student - Enrollment - Course: A student can enroll in multiple courses, and each course can have many students (Many-to-Many).
  • Instructor - Course: An instructor can teach many courses (One-to-Many).
  • Course - Schedule: A course can have multiple schedules for different times and days (One-to-Many).

üìö Final Thoughts

The design outlined above ensures that data related to courses, students, and instructors is stored efficiently and can be easily queried. The relationships between tables are clear and help maintain data integrity.
