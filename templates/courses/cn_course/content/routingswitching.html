Routing & Switching

Routing and Switching are the core functions that allow data to move across computer networks. Switching connects devices within the same network, while Routing connects different networks together.

üîÄ What is Switching?

Switching is the process of forwarding data frames between devices within the same network (LAN). Switches operate at the Data Link Layer (Layer 2) of the OSI model and use MAC addresses to make forwarding decisions.

üîé Key Functions of Switching

  • MAC Address Learning: Switches learn the MAC addresses of devices connected to their ports.
  • Frame Forwarding: Based on MAC address tables, switches forward frames only to the correct destination port.
  • Loop Prevention: Using protocols like STP (Spanning Tree Protocol), switches prevent loops that can crash the network.

üìä Types of Switching

  • Cut-Through Switching: Starts forwarding frames as soon as the destination MAC address is read, reducing latency.
  • Store-and-Forward Switching: Receives the entire frame, checks for errors, and then forwards it. More reliable but slower.

‚ö° Benefits of Switching

  • Reduces network congestion by creating separate collision domains per port.
  • Improves performance by forwarding frames only to intended recipients.
  • Supports VLANs (Virtual LANs) for segmenting networks logically.

üåê What is Routing?

Routing is the process of forwarding data packets between different networks. Routers operate at the Network Layer (Layer 3) and use IP addresses to make forwarding decisions.

üîé Key Functions of Routing

  • Path Selection: Routers choose the best path for forwarding packets based on routing tables.
  • Packet Forwarding: Uses destination IP address to forward packets to next hop or final destination.
  • Inter-Network Communication: Connects multiple LANs or WANs together, enabling global communication.

üìö Types of Routing

  • Static Routing: Routes are manually configured by network admins. Simple but doesn‚Äôt adapt to changes automatically.
  • Dynamic Routing: Uses protocols (like OSPF, EIGRP, RIP) to automatically adjust routes based on network changes.

üöÄ Common Routing Protocols

Protocol Description
RIP (Routing Information Protocol) Uses hop count as metric, simple but not suitable for large networks.
OSPF (Open Shortest Path First) Link-state protocol, scalable and widely used in enterprises.
EIGRP (Enhanced Interior Gateway Routing Protocol) Hybrid protocol used mainly in Cisco networks, combines best of RIP and OSPF.
BGP (Border Gateway Protocol) Used to route between different organizations (Internet routing protocol).

üîß VLANs (Virtual LANs)

VLANs are used in switches to segment a physical network into multiple logical networks. Devices in different VLANs cannot communicate without a router.

  • Improves security by isolating sensitive data.
  • Optimizes performance by reducing broadcast traffic.
  • Requires Layer 3 routing to connect between VLANs.

üö¶ Inter-VLAN Routing

To allow communication between VLANs, routers (or Layer 3 switches) perform Inter-VLAN Routing. This enables devices in different VLANs to talk to each other.

  • Router-on-a-Stick: A single router interface handles multiple VLANs using sub-interfaces.
  • Layer 3 Switches: Can route between VLANs without needing an external router.

‚ùì Common Questions

Q: Can a switch perform routing?

A: Yes, Layer 3 switches can perform routing functions between VLANs or networks.

Q: Why do we use routers instead of just switches?

A: Switches connect devices in the same network, but routers are needed to connect different networks (e.g., LAN to the Internet).

Q: What is dynamic routing?

A: Dynamic routing uses protocols to automatically update routing tables based on changes in the network topology.

üéØ Pro Tip

Remember: Switches use MAC addresses (Layer 2), Routers use IP addresses (Layer 3). VLANs need routing to communicate across each other.
