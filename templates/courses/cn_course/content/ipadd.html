IP Addressing & Subnetting

IP addressing and subnetting are fundamental concepts in computer networking that define how devices communicate over the internet and private networks.

🔎 What is an IP Address?

An IP address (Internet Protocol address) is a unique identifier assigned to each device connected to a network. It allows devices to find and communicate with each other across local and global networks.

Example: 192.168.1.10

📚 Types of IP Addresses

  • IPv4: 32-bit address, written in dotted decimal format (e.g., 192.168.0.1). Supports ~4.3 billion addresses.
  • IPv6: 128-bit address, written in hexadecimal (e.g., 2001:0db8:85a3::8a2e:0370:7334). Supports trillions of devices.

📊 IPv4 Address Classes

Class Range Default Subnet Mask Usage
Class A 1.0.0.0 to 126.255.255.255 255.0.0.0 Large networks
Class B 128.0.0.0 to 191.255.255.255 255.255.0.0 Medium networks
Class C 192.0.0.0 to 223.255.255.255 255.255.255.0 Small networks
Class D 224.0.0.0 to 239.255.255.255 N/A Multicast
Class E 240.0.0.0 to 255.255.255.255 N/A Experimental

🏠 Private vs Public IP Addresses

  • Private IPs: Used within local networks (e.g., 192.168.x.x, 10.x.x.x). Not routable on the internet.
  • Public IPs: Unique globally and used to identify devices on the internet.

📏 What is Subnetting?

Subnetting is the process of dividing a large network into smaller, manageable sub-networks (subnets). It helps in efficient IP address allocation and improves network security and performance.

📚 Subnet Mask Explained

A subnet mask separates the network portion from the host portion in an IP address. Example:

  • IP Address: 192.168.1.10
  • Subnet Mask: 255.255.255.0
  • Network Portion: 192.168.1
  • Host Portion: .10

🧮 CIDR Notation

CIDR (Classless Inter-Domain Routing) is a more flexible method of representing subnet masks.

  • /24 = 255.255.255.0
  • /16 = 255.255.0.0
  • /8 = 255.0.0.0

📊 Common Subnets

CIDR Subnet Mask Usable Hosts

✅ Subnetting Example

Let’s subnet the network 192.168.1.0/24 into 4 subnets:

  • New Subnet Mask: /26 (255.255.255.192)
  • Each subnet has 62 usable hosts.
  • Subnets:
    • 192.168.1.0/26 → Hosts: 192.168.1.1 to 192.168.1.62
    • 192.168.1.64/26 → Hosts: 192.168.1.65 to 192.168.1.126
    • 192.168.1.128/26 → Hosts: 192.168.1.129 to 192.168.1.190
    • 192.168.1.192/26 → Hosts: 192.168.1.193 to 192.168.1.254

🔄 How Subnetting Helps

  • Efficient IP address utilization
  • Improves network performance by reducing broadcast traffic
  • Enhances security by isolating network segments

❓ Common Questions

Q: Why is subnetting important?

A: It allows better management of IP addresses, improves security, and reduces congestion in large networks.

Q: What’s the difference between public and private IPs?

A: Private IPs work within local networks; public IPs are used globally on the internet.

Q: Can I subnet a Class C network?

A: Yes! For example, a Class C /24 can be split into multiple smaller subnets like /26 or /27.

🎯 Pro Tip

Use the formula 2^n - 2 (where n = host bits) to calculate usable hosts in a subnet.
