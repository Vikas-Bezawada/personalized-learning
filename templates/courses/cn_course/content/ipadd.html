IP Addressing & Subnetting

IP addressing and subnetting are fundamental concepts in computer networking that define how devices communicate over the internet and private networks.

ğŸ” What is an IP Address?

An IP address (Internet Protocol address) is a unique identifier assigned to each device connected to a network. It allows devices to find and communicate with each other across local and global networks.

Example: 192.168.1.10

ğŸ“š Types of IP Addresses

  • IPv4: 32-bit address, written in dotted decimal format (e.g., 192.168.0.1). Supports ~4.3 billion addresses.
  • IPv6: 128-bit address, written in hexadecimal (e.g., 2001:0db8:85a3::8a2e:0370:7334). Supports trillions of devices.

ğŸ  Private vs Public IP Addresses

  • Private IPs: Used within local networks (e.g., 192.168.x.x, 10.x.x.x). Not routable on the internet.
  • Public IPs: Unique globally and used to identify devices on the internet.

ğŸ“ What is Subnetting?

Subnetting is the process of dividing a large network into smaller, manageable sub-networks (subnets). It helps in efficient IP address allocation and improves network security and performance.

ğŸ“š Subnet Mask Explained

A subnet mask separates the network portion from the host portion in an IP address. Example:

  • IP Address: 192.168.1.10
  • Subnet Mask: 255.255.255.0
  • Network Portion: 192.168.1
  • Host Portion: .10

ğŸ§® CIDR Notation

CIDR (Classless Inter-Domain Routing) is a more flexible method of representing subnet masks.

  • /24 = 255.255.255.0
  • /16 = 255.255.0.0
  • /8 = 255.0.0.0
