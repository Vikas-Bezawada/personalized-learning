Networking Protocols

Protocols are rules that define how data is transmitted over networks. They ensure devices can communicate effectively, securely, and reliably.

üåê 1. HTTP (HyperText Transfer Protocol)

HTTP is the foundation of data communication on the web. It is used to load webpages by transferring hypertext between a client (browser) and server.

  • Uses port 80.
  • Stateless protocol (each request is independent).
  • Not secure (data is sent in plain text).

üîí 2. HTTPS (HTTP Secure)

HTTPS is the secure version of HTTP. It encrypts the data using SSL/TLS to protect against interception.

  • Uses port 443.
  • Encrypts communication between browser and server.
  • Essential for e-commerce, banking, and secure logins.

üì¶ 3. FTP (File Transfer Protocol)

FTP is used to transfer files between computers over a network. It allows uploading and downloading files from servers.

  • Uses ports 20 (data) and 21 (control).
  • Does not encrypt data (use SFTP/FTPS for secure transfer).
  • Commonly used in website management and file sharing.

üìß 4. SMTP (Simple Mail Transfer Protocol)

SMTP is used to send emails from clients to servers and between mail servers.

  • Uses port 25 (unencrypted) or 587 (encrypted).
  • Handles sending emails, not receiving.
  • Works with POP3/IMAP for complete email service.

üì• 5. POP3 (Post Office Protocol version 3)

POP3 is used by email clients to retrieve emails from a server. It downloads and usually deletes mail from the server.

  • Uses port 110 (unencrypted) or 995 (secure).
  • Emails are stored locally after download.
  • Simple but doesn't sync across multiple devices.

üìß 6. IMAP (Internet Message Access Protocol)

IMAP is a protocol that allows email clients to access and manage mail on the server without downloading.

  • Uses port 143 (unencrypted) or 993 (secure).
  • Emails stay on the server and sync across devices.
  • Preferred for modern email apps (Gmail, Outlook, etc.).

üì° 7. DNS (Domain Name System)

DNS translates human-readable domain names (like google.com) into IP addresses that computers understand.

  • Uses port 53.
  • Acts like the internet's "phonebook."
  • Vital for browsing websites by name.

üì® 8. DHCP (Dynamic Host Configuration Protocol)

DHCP automatically assigns IP addresses and network configuration to devices on a network.

  • Uses ports 67 (server) and 68 (client).
  • Simplifies network management by avoiding manual IP assignment.
  • Common in home routers and enterprise networks.

üìä 9. SNMP (Simple Network Management Protocol)

SNMP is used to monitor and manage network devices like routers, switches, and servers.

  • Uses port 161.
  • Collects performance and usage data from devices.
  • Used in network monitoring tools.

üí¨ 10. Telnet

Telnet is an old protocol used to remotely access devices via command-line. It's insecure because data is transmitted in plain text.

  • Uses port 23.
  • Replaced by SSH for secure remote access.
  • Still used in testing environments.

üîê 11. SSH (Secure Shell)

SSH is a protocol for secure remote login and command execution on network devices and servers.

  • Uses port 22.
  • Encrypts data, unlike Telnet.
  • Widely used by system administrators and developers.

üì∂ 12. NTP (Network Time Protocol)

NTP synchronizes clocks of computers and devices across a network to ensure time consistency.

  • Uses port 123.
  • Critical for logs, security, and scheduling tasks.
  • Used by servers, routers, and security appliances.

‚úÖ 13. ARP (Address Resolution Protocol)

ARP resolves IP addresses to MAC (hardware) addresses on a local network. It's crucial for LAN communication.

  • Operates at the network layer (Layer 3).
  • Enables devices to locate each other on the same network.
  • Used in switches and routers for routing decisions.

Q: Why is SSH preferred over Telnet?
