Collections & Streams

Introduction: Collections in Java are frameworks that provide a way to store and manipulate groups of data. Streams provide a way to process sequences of elements (e.g., from a collection) in a functional style.

Collections Framework: The Collections Framework provides various interfaces, implementations, and algorithms to store and manipulate data. Some important interfaces include:

  • List: An ordered collection that allows duplicate elements. Examples: `ArrayList`, `LinkedList`.
  • Set: A collection that does not allow duplicate elements. Examples: `HashSet`, `TreeSet`.
  • Map: A collection of key-value pairs. Examples: `HashMap`, `TreeMap`.

ArrayList Example: The `ArrayList` class implements the `List` interface and allows dynamic resizing of arrays.

import java.util.ArrayList;

public class ArrayListExample {
    public static void main(String[] args) {
        ArrayList list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("C++");

        // Accessing elements by index
        System.out.println("First element: " + list.get(0));

        // Removing an element
        list.remove("Python");

        // Checking if an element exists
        if (list.contains("C++")) {
            System.out.println("C++ is in the list.");
        }

        System.out.println("List: " + list);
    }
}

LinkedHashSet Example: The `LinkedHashSet` class implements the `Set` interface and maintains the insertion order. It does not allow duplicate values.

import java.util.LinkedHashSet;

public class LinkedHashSetExample {
    public static void main(String[] args) {
        LinkedHashSet set = new LinkedHashSet<>();
        set.add("Java");
        set.add("Python");
        set.add("Java");  // Duplicate element, will not be added

        System.out.println("LinkedHashSet (insertion order): " + set);
    }
}

Set Example: The `HashSet` class implements the `Set` interface and does not allow duplicate values.

import java.util.HashSet;

public class SetExample {
    public static void main(String[] args) {
        HashSet set = new HashSet<>();
        set.add("Java");
        set.add("Python");
        set.add("Java");  // Duplicate element, will not be added
        
        System.out.println("Set: " + set);
    }
}

Map Example: The `HashMap` class implements the `Map` interface and stores key-value pairs.

import java.util.HashMap;

Streams: Streams provide a functional approach to processing data. They allow you to perform operations such as filtering, mapping, and reducing in a concise and expressive way.

Stream Example: A simple example of filtering even numbers from a list using streams.

import java.util.Arrays;
import java.util.List;
import java.util.stream.Collectors;

public class StreamExample {
    public static void main(String[] args) {
        List numbers = Arrays.asList(1, 2, 3, 4, 5, 6);
        
        List evenNumbers = numbers.stream()
                                           .filter(n -> n % 2 == 0)  // Filter even numbers
                                           .collect(Collectors.toList());
        
        System.out.println("Even Numbers: " + evenNumbers);
    }
}

Stream Operations: Streams support various intermediate and terminal operations. Some common operations include:

  • filter: Filters elements based on a condition.
  • map: Transforms elements using a function.
  • reduce: Aggregates elements into a single result.
  • forEach: Performs an action on each element of the stream.

Example of map and reduce: This example multiplies each number by 2 and then sums them up.

import java.util.Arrays;
import java.util.List;

public class MapReduceExample {
    public static void main(String[] args) {
        List numbers = Arrays.asList(1, 2, 3, 4, 5);
        
        int sum = numbers.stream()
                         .map(n -> n * 2)  // Multiply each number by 2
                         .reduce(0, Integer::sum);  // Sum the results
        
        System.out.println("Sum of doubled numbers: " + sum);
    }
}

Additional Tip: Streams make your code more readable and concise, especially when dealing with collections and performing multiple operations on data.
