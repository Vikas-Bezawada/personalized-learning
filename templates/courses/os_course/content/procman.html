Process Management in Operating Systems

Process Management is one of the core functions of an Operating System. It is responsible for creating, scheduling, and terminating processes, as well as handling communication and synchronization between them.

üîé What is a Process?

A process is an instance of a program in execution. It includes the program code, current activity (values in registers, program counter), stack, heap, and data section. Processes are the active entities, whereas programs are passive.

üìä Process States

State Description
New The process is being created.
Ready Process is waiting to be assigned to the CPU.
Running Process instructions are being executed by the CPU.
Waiting (Blocked) Process is waiting for some event (like I/O completion).
Terminated Process has finished execution.

üìù Process Control Block (PCB)

Every process is represented in the OS by a Process Control Block (PCB), which contains crucial information:

  • Process ID (PID): Unique identifier for the process
  • Process State: Current state (ready, running, waiting)
  • Program Counter: Address of the next instruction
  • CPU Registers: Values that the process is using
  • Memory Management Info: Page tables, segment tables
  • I/O Status: List of I/O devices allocated to the process

‚öôÔ∏è Process Scheduling

The OS uses schedulers to decide which process gets to use the CPU next. This is key to efficient multitasking.

Types of Schedulers

Scheduler Description
Long-Term Scheduler Decides which processes are admitted to the system for processing.
Short-Term Scheduler Selects from among the ready processes and allocates the CPU.
Medium-Term Scheduler Removes processes from memory (swapping) to reduce load.

üîÑ Context Switching

Context Switching is the mechanism where the OS saves the context (state) of a running process and loads the context of another process. It allows multiple processes to share a single CPU.

  • Stores the current process's PCB
  • Loads the next process's PCB
  • Resumes execution of the next process

üîó Inter-Process Communication (IPC)

Processes often need to communicate and synchronize. IPC mechanisms include:

  • Pipes - for communication between related processes
  • Message Queues - for sending and receiving messages
  • Shared Memory - multiple processes access the same memory area
  • Sockets - for network communication between processes

üìö Example Process Commands

  • List running processes: ps -aux
  • Kill a process by PID: kill 1234
  • View process tree: pstree

‚ùì Common Questions

Q: What's the difference between a process and a thread?

A: A process is an independent execution unit with its own memory space, while a thread is a lightweight process sharing memory within a process.

Q: Why is context switching expensive?

A: Context switching involves saving and loading register states and memory maps, which takes CPU time and slows down performance.

Q: Can multiple processes run at the same time?

A: Yes, on multicore systems, multiple processes can execute truly in parallel. On single-core systems, multitasking is simulated using context switching.
